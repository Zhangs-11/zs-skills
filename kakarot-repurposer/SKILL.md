# kakarot-repurposer 一稿多平台改写

你正在帮「卡卡罗特学AI」做内容分发：把一篇已经写好的公众号长文，改写成**小红书笔记**和**抖音脚本**两种形态，让一篇内容在三个平台都能用。

## 一句话讲清这个 skill 在干嘛

公众号文章 = 给人**读**的长文。小红书 = 给人**刷**的短图文。抖音 = 给人**看/听**的短视频稿。

三个平台的读者、节奏、耐心完全不同。**不是把长文复制粘贴过去，也不是机械缩写，而是用各平台的语言把同一件事重新讲一遍。** 核心那个「打动人的点」不变，表达方式全变。

> ⚠️ 最容易犯的错：把公众号文章直接砍短就当小红书发。砍出来的东西又干又硬，没有平台味，必死。每一版都要按目标平台重新组织。

## 输入

用户会给你一篇公众号文章，可能是：
- `~/公众号草稿/` 下的某个 `.md` 文件（最常见）
- 直接粘贴的正文
- 一个文章主题 + 要点

**先通读全文，找出三样东西**，这是所有改写的原料：
1. **一个最锋利的钩子**（最反常识 / 最戳情绪的那一句或那个点）
2. **2-4 个干货点**（读者真能学到/带走的东西）
3. **作者的真实声音**（哪句「我觉得」「我当时就愣住了」最有活人感）

## 保持卡卡罗特的活人感（最重要的底线）

不管改成哪个平台，下面这些是「卡卡罗特」的灵魂，**绝不能在改写中丢掉**：

- **讲人话，像个活人。** 大胆用「我觉得」「我试了下」「说实话」。不要变成冷冰冰的知识播报。
- **真诚。** 有缺点就说缺点，不懂就说不懂。不浮夸、不震惊体。
- **好奇心打底。** 出发点永远是「这玩意儿好有意思」，不是「你不知道就完了」。
- **不说教。** 是分享，是「我跟你唠唠我发现的事」，不是「我来教育你」。

> 改写时心里有个判断标准：这话像不像一个真实的、有点东西的年轻人会说的？像，就对了；像营销号 / 像 AI 写的，就推翻重来。

# 一、小红书笔记版

## 小红书是什么调性

小红书是**「真诚分享 + 即时有用」**的地方。用户在这刷的是「真人的经验帖」。爆款笔记的底层是：**一眼能看懂、马上能用上、感觉是真人在跟我说话。**

## 小红书笔记的结构

```
【标题】20 字以内，带钩子，可配 1-2 个 emoji
【正文】
  开头 1-2 句：直接抛出最反常识/最戳人的点（不铺垫，小红书没耐心）
  中间：2-4 个干货点，每点一小段，多空行，可用 emoji 当小标题
  结尾：一句真诚的个人感受 / 一个轻互动（你们怎么看？）
【话题标签】5-10 个 #标签，混合大词 + 精准词
```

## 小红书改写规则

- **标题**：放最锋利的那个钩子。形式可以是疑问（「AI 突然变聪明了？就因为偷偷做了这件事」）、反差（「我以为是玄学，结果是工程」）、数字（「3 个变化，看懂大模型这一年」）。**别用公众号那种长问句标题，小红书要更短更冲。**
- **正文**：
  - 句子砍短，一句话一个意思，**多换行多空行**，手机上一屏不能太密。
  - 适度用 emoji 当**视觉锚点**（✅ 🔥 👇 💡），但别堆成表情包。
  - 干货点用「人话」讲，能用比喻就用比喻。
  - 保留 1-2 处作者的真实声音，让它有体温。
- **长度**：300-800 字，比公众号短得多。只留最硬的干货，啰嗦的论证全砍。
- **话题标签**：5-10 个。大词（#人工智能 #AI）保流量 + 精准词（#大模型 #AI学习）保精准 + 一个账号词（#卡卡罗特学AI）。
- **配图建议**：给出封面图该写什么大字（小红书封面靠大字抓人）+ 2-3 张内容图建议。

## 小红书输出模板

```markdown
## 📕 小红书笔记版

**标题：** （20字内，带钩子）

**正文：**
（开头钩子句）

（干货点 1，配 emoji 锚点）

（干货点 2）

（干货点 3）

（结尾个人感受 + 轻互动）

**话题标签：** #标签1 #标签2 #标签3 #标签4 #标签5 #卡卡罗特学AI

**配图建议：**
- 封面：建议大字写「XXX」，背景 XXX
- 内容图：1）XXX  2）XXX
```

# 二、抖音脚本版

## 抖音是什么调性

抖音是**「黄金 3 秒 + 强节奏」**的地方。前 3 秒抓不住人就被划走。内容要短、要有钩子、要有「看下去的理由」。

## 一份稿，三种用法（重点）

抖音脚本我做成一份**分句脚本**——**一句一行，每行就是一个画面单元**。这一份稿同时支持三种制作方式，用户不用提前决定：

1. **纯文字卡点**：每行直接当字幕，配 BGM，不用人声。
2. **AI 配音**：整段文字粘进**剪映「文本朗读」**，一键生成人声 + 自动字幕（最省事，免费，几十种音色）。
3. **真人口播**：照着念。

所以脚本要满足：**每一句都短到能一口气念完、也能一屏看完。**

## 抖音脚本的结构

```
【黄金3秒钩子】1-2 句，必须制造好奇/反差/痛点，让人停下来
【正文口播】把 2-4 个干货点用大白话讲，一句一行，句子短
【转折/高潮】一个「但是…」「然而…」制造信息差，留住人看到最后
【结尾引导】一句话总结 + 引导关注（别生硬，用「下条接着讲」这种）
```

## 抖音改写规则

- **黄金 3 秒**：开头那句要么戳痛点（「你是不是也发现…」），要么抛反差（「我一直以为 X，结果是 Y」），要么给悬念（「今天讲个很多人不知道的事」）。**绝不能慢慢铺垫。**
- **句子**：每句短，口语化，能念顺。删掉所有书面连接词（「综上所述」「因此」），换成说话的语气（「所以你看」「这就有意思了」）。
- **节奏**：一句一行。可以用「。。。」制造停顿感（卡卡罗特的标志性停顿）。
- **长度**：300-600 字，念出来 1-2 分钟。
- **结尾**：自然引导关注/追更，别喊「快关注我」这种硬广。
- **附带标注**：给每个段落配**画面提示**（拍什么/放什么图）、**BGM 建议**、关键句的**语气提示**（可选）。

## 抖音输出模板

```markdown
## 🎬 抖音脚本版（一句一行，三种用法通用）

> 用法：纯文字→每行当字幕｜AI配音→整段粘进剪映「文本朗读」｜真人→照着念

**【黄金3秒钩子】**
（第一句，画面：XXX）
（第二句）

**【正文】**
（干货句 1）
（干货句 2）
（干货句 3）

**【转折】**
（一个「但是…」制造信息差）

**【结尾引导】**
（总结句 + 自然引导关注）

---
**画面/BGM 建议：**
- 开头：画面 XXX，BGM 用 XXX 风格（紧凑/轻快）
- 中段：配 XXX 图
- 标题大字建议：「XXX」
```

# 四、交付：一份完整素材包

每篇生成一个**素材包**目录 `~/公众号草稿/分发/<文章名>/`，目标是让用户最后只剩「粘贴 + 存草稿」这一下。包含：

| 文件 | 说明 |
|------|------|
| `xiaohongshu.md` | 小红书笔记（标题 / 正文 / 话题标签 / 配图建议） |
| `douyin.md` | 抖音脚本（口播稿 + 画面/BGM 建议） |
| `voice.txt` | 抖音口播**纯念稿**（一句一行，去掉所有画面/段落标记，只留要念的字） |
| `voice.m4a` | 抖音 AI 配音 |
| `subtitles.srt` | 抖音字幕（可直接导入剪映，不用手打） |
| `preview.html` | 效果预览页（仅审稿用，**不是发布通道**） |
| `发布清单.md` | 三平台手动发布步骤清单 |

配图复用原文章 `~/公众号草稿/images/` 下已有的图。

## 生成配音（macOS）
把口播稿落成 `voice.txt`（一句一行），用系统中文音色生成配音：
```bash
cd ~/公众号草稿/分发/<文章名>
say -f voice.txt -v Tingting -r 190 -o voice.aiff
afconvert voice.aiff voice.m4a -f m4af -d aac
```
`Tingting` = 普通话女声（另有 Meijia 台普、Sinji 粤语）。正式出片可改用剪映「文本朗读」挑更多音色。

## 生成字幕
```bash
python3 <skill目录>/scripts/make_subtitles.py \
  --voice-txt voice.txt --audio voice.m4a --out subtitles.srt
```
按每句字数占比把配音时长切段，输出标准 `.srt`。

## 预览页 preview.html
- 左：小红书 iPhone 笔记排版（套真实配图）。
- 右：抖音 9:16 竖屏，**单屏字幕原地切换**（画面固定不动，配音念到哪句字幕换到哪句；**绝不要做成整屏上滑**，那看着像刷下一条视频），带 ▶️ 播放真实配音 + 字幕跟随高亮 + 顶部进度圆点。
- 生成后 `open preview.html` 给用户审稿，**先看效果、确认后再发布**。

## 落地：备齐素材，手动粘（当前选定方式）
- **公众号** → wechat-publisher 自动发草稿箱。
- **小红书** → 电脑开 creator.xiaohongshu.com → 发布笔记 → 粘 `xiaohongshu.md` + 传图 → 存草稿。
- **抖音** → 剪映 → 文本朗读 / 导入 `voice.m4a` + `subtitles.srt` → 同步抖音草稿。

> ⚠️ 小红书/抖音**没有面向个人的官方草稿箱 API**，最后一步必须人工粘贴或借第三方工具（蚁小二、新榜矩阵通）。**永远不要承诺「自动发到小红书/抖音草稿箱」——做不到，那是微信公众号才有的特例。**

# 六、发布前质检

每个平台版本交付前自查：

1. **没有「翻译腔」**：读起来是不是这个平台的真实笔记/视频？还是一眼能看出是长文砍出来的？后者重写。
2. **活人感还在不在**：至少保留一处作者的真实声音，没变成冷冰冰的知识播报。
3. **钩子够不够冲**：小红书标题/抖音前 3 秒，能不能让人停下来？
4. **能不能直接用**：小红书排版手机上一屏顺不顺？抖音每句能不能一口气念完？
5. **真诚不浮夸**：没有震惊体、没有「不看后悔」这种营销号话术。
